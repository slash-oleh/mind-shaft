# General: Naming

## Avoid technical suffixes
Avoid embedding implementation details or generic roles like 'Object', 'String', 'Helper' in identifiers and use plural forms instead of suffixes for collections.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/avoid-technical-suffixes.md)

## Care about variable context: not too generic and not too specific
Ensure variable names provide sufficient information for clarity while avoiding redundant details already provided by the surrounding context.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/care-about-variable-context-not-too-generic-and-not-too-specific.md)

## Standardize common terms
Use a single, project-wide term for common concepts and operations names.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/standardize-common-terms.md)

## Use consistent casing
Use consistent casing for all variable and function names, and treat acronyms (e.g., `userId`) as regular words to maintain clear word boundaries.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/use-consistent-casing.md)

## Use content-agnostic translation keys
Use descriptive translation keys that represent the purpose or context of a message rather than its literal content.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/use-content-agnostic-translation-keys.md)

## Use full and concise names
Use complete, descriptive words instead of cryptic abbreviations while avoiding excessive verbosity that doesn't add clarity.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/use-full-and-concise-names.md)

## Use nouns for data and verbs for ops
Use nouns for data variables and verbs for functions or methods to clearly distinguish between data and operations.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/use-nouns-for-data-and-verbs-for-ops.md)

## Use positive naming
Use positive identifiers for variables and states instead of inverted logic.
[read more](https://github.com/insolite/dev-rules/blob/main/src/general/naming/use-positive-naming.md)
