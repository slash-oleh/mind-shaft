# Languages: Typescript

## Export associated types
Export parameter interfaces and type aliases alongside their functions.
[read more](https://github.com/insolite/dev-rules/blob/main/src/languages/typescript/export-associated-types.md)

## Reuse existing types
Reuse existing internal and third-party types through inheritance or imports instead of re-declaring their structure.
[read more](https://github.com/insolite/dev-rules/blob/main/src/languages/typescript/reuse-existing-types.md)

## Use explicit types instead of type assertions
Explicitly annotate variables and function returns with types instead of relying on inference for complex objects or using type assertions to force compliance.
[read more](https://github.com/insolite/dev-rules/blob/main/src/languages/typescript/use-explicit-types-instead-of-type-assertions.md)

## Use inheritance for hierarchical types
Use interface extension (`extends`) to model hierarchical relationships and share common properties instead of defining unions of types that duplicate base fields.
[read more](https://github.com/insolite/dev-rules/blob/main/src/languages/typescript/use-inheritance-for-hierarchical-types.md)
