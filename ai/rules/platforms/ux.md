- **Add explanation to disabled controls**: Provide a visible explanation of *why* a control is disabled and *what* the user can do to enable it.
- **Avoid floating areas**: Minimize the use of modal windows, pop‑overs, and other floating UI elements.
- **Don't implement unjustifiably complex design traits without clarifying**: Clarify unusually complex or inconsistent design details with the designer before implementation.
- **Don't wrap interactive elements into each other**: Avoid nesting interactive elements (buttons, links, inputs) inside other interactive elements.
- **Limit component variations**: Standardize on a small set of well-defined component variants. Avoid creating multiple components or variations that serve the same underlying purpose.
- **Limit simultaneous components**: Limit the number of interactive elements, data points, and navigation options displayed simultaneously.
- **Limit the use of color**: Use color sparingly and purposefully. A restrained color palette improves clarity, focuses attention, and makes meaningful color cues (like errors or success states) stand out more effectively.
- **Maintain core accessibility features**: Preserve and enhance native browser accessibility features.
- **Make form fail early**: Validate form inputs as soon as the user interacts with them instead of waiting for submission.
- **Prefer fail-tolerant approaches**: Design interfaces that prevent errors before they happen and allow users to recover easily.
- **Prefer flat menus over nested submenus**: Keep navigation shallow by using flat lists with clear grouping instead of nested submenus.
- **Use high-visibility controls for few options**: Use button groups, switches, or radio buttons instead of dropdowns for small sets of options (typically 2-5).
- **Use visual accents**: Establish a clear visual hierarchy using variations in font weight, color, size, and positioning to guide user attention.
