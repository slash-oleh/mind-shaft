# Dev Handbook

Welcome to the centralized knowledge base for software development excellence. This handbook provides actionable guidance, technical standards, and skill development resources for engineering teams.

## Content Sections

### [Rules](./rules)

Atomic best practices, design principles, and architectural standards, coding preferences. Each rule is formulated as a problem-solution.

### [Skills](./skills)

Processes, flows, aggregated knowledge items for different repeated workflows. Mainly focused to be used via AI agents.

### [Stack](./stack)

Recommendations about technology stack. Including tools, services, libraries. Collections and decision trees.

---

## How to use it

1. **Learn**: Use the navigation/search to explore specific topics yourself.
2. **Feed AI**: Source `ai` repo folder with tailored TLDR version of everything into your agent to make it follow this handbook.
3. **Reference**: Use it as an external source to deduplicate your team/project contribution guidelines, code reviews.
4. **Contribute**: If you find a gap or want to improve a rule, check the contribution guidelines in the repo.
