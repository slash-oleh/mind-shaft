# Phase 2: Plan

## Goal

An ordered, step-by-step implementation plan detailing exact file changes, commits, and risk flags is established.

## Steps

### Step 1: Structure Logical Steps

For each sub-task:

- Target single coherent changes.
- Name exact files and describe specific changes.
- Ensure small change (single commit size).

Ensure foundation first (types, models), then utilities, then consumers/UI.

## Output

Markdown format:

- Implementation Steps: Ordered list of steps with files, changes, and dependencies.
