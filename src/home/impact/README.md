# Impact
 
All characteristics (the "why" part) are described through the **Impact**. Impact items are argued separately from recommendation items.
Every impact item can imply either positive or negative consequences.
 
The articles should avoid using ambiguous characteristics such as words:
*better*, *improved*, *increased*, *flexible*, *obvious*, *explicit*, *easy*, etc.
without definition or arguing.
 
We also don't use these as Impact:
 
- Speed (Velocity)
- Quality
- Cost (Price)
 
Impact items should be more concrete, rather be **ways** to achieve these as goals.

What is also not a Impact:

- Tech Debt - goes automatically with any bad solution we decide to keep
