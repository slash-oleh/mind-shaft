# Quality Attributes

All characteristics (the "why" part) is described through the **Quality
Attributes**. Quality attributes are argued separately from recommendation item.
Every quality attribute can imply either positive or negative impact.

The articles should avoid using ambiguous characteristics such as words:
*better*, *improved*, *increased*, *flexible*, *obvious*, *explicit*, *easy*, etc.
without definition or arguing.

We also don't use these as quality attributes:

- Speed (Velocity)
- Quality
- Cost (Price)

Quality Attributes should be more concrete, rather be **ways** to achieve these as goals.

What is also not a Quality Attribute:

- Tech Debt - goes automatically with any bad solution we decide to keep
